## 슬라이드 2 — 바이브 코딩 3원칙 (5분)

### 1. 의도 먼저

- 기술 용어 없어도 됩니다
- "파란 버튼 만들어줘"가 시작입니다

### 2. 작게 이터레이션

- 한 번에 완성하려 하지 마세요
- 한 기능씩 확인하며 쌓으면 됩니다

### 3. 결과물 책임은 나

- AI가 짠 코드도 내 코드입니다
- 실행해보고 이상하면 다시 시킵니다

**기존 개발과 다른 점**: `타이핑 → 실행` 에서 `대화 → 확인` 루프로 전환

## 슬라이드 3 — 프롬프트 잘 쓰는 법

### 공식: 맥락 + 원하는 동작 + 결과 형태

| 나쁜 프롬프트   | 좋은 프롬프트                                                        |
| --------------- | -------------------------------------------------------------------- |
| 버튼 만들어줘   | 오른쪽 하단에 파란 CTA 버튼, 클릭 시 팀 매칭 시작되게 해줘           |
| 예쁘게 만들어줘 | 카드 컴포넌트에 hover 시 그림자 효과와 scale(1.02) 트랜지션 추가해줘 |
| 배포해줘        | vercel로 프로덕션 배포하고 나온 URL 알려줘                           |

**에러가 났을 때도 당황 금지** — 에러 메시지를 그대로 붙여넣으면 됩니다.

이 원칙은 비개발자 도구든 개발자 도구든 동일하게 적용됩니다.

## 슬라이드 4 — DEMO: 비개발자 편 — v0로 딸깍 배포 (10분)

### 먼저 보여드리겠습니다 — 코드 없이 배포까지

### 순서

1. **v0.app 접속**: 브라우저에서 v0.app 열기
2. **프롬프트 입력**: _"팀 매칭 서비스 메인 페이지 만들어줘. 카드 리스트와 매칭 버튼 포함"_
3. **결과 확인**: 생성된 UI 확인 → 마음에 안 드는 부분 수정 요청
4. **수정 요청**: _"카드에 hover 효과 추가하고, 색상을 파란 계열로 바꿔줘"_
5. **딸깍 배포**: Publish 버튼 클릭 → 라이브 URL 생성

### 핵심 멘트

> "지금 터미널을 한 번도 안 열었습니다. 코드도 한 줄도 안 썼습니다. 그런데 배포까지 끝났어요."
> "이게 바이브 코딩입니다. 이제 해커톤에서 직접 해보세요."

### 비개발자 도구 비교 (말로 보충) [^1]

| 도구         | 특징                                        | 배포                 |
| ------------ | ------------------------------------------- | -------------------- |
| v0 by Vercel | 프롬프트로 UI 생성 (오늘 시연)              | Publish 버튼 딸깍    |
| Lovable      | 자연어로 앱 생성, DB/인증 자동 연결         | 원클릭 배포          |
| Bolt.new     | 브라우저 안에서 코드 편집 + 실시간 미리보기 | 내장 배포            |
| Claude.ai    | 채팅으로 코드 생성                          | 복사 붙여넣기로 활용 |

## 슬라이드 5 — 그런데 개발자는 더 깊이 갈 수 있습니다 (5분)

### 방금 본 v0로는 어려운 것들

- 세밀한 코드 제어와 커스터마이징
- 외부 API/서비스 연동 (MCP) [^3]
- 기존 프로젝트에 기능 추가
- 복잡한 비즈니스 로직 구현

### 개발자 도구 파이프라인

```
Google Stitch  →  stitch-mcp (브릿지)  →  Claude Code  →  Vercel  →  라이브 URL
 텍스트→UI         MCP로 디자인 전달       Next.js 변환     30초 배포     전 세계 접근
```

MCP 등록 명령어: [^2]

```bash
claude mcp add stitch --transport http https://stitch.googleapis.com/mcp \
  --header "X-Goog-Api-Key: YOUR-API-KEY" -s user
```

### 핵심 멘트

> "방금 v0로 배포한 것과 같은 결과를, 이번엔 개발자 도구로 해보겠습니다. 차이점을 보세요."

## 슬라이드 7 — DEMO: 개발자 편 2 — Vercel CLI 배포 (5분)

### 순서

1. **로컬 확인**: `npm run dev` → 브라우저에서 결과 미리보기
2. **Vercel 배포**: `vercel` → 프로젝트명 엔터 → 빌드 시작 [^4]
3. **URL 획득**: `https://xxx.vercel.app` — 30초 안에 라이브 URL 등장
4. **청중 공유**: 채팅창에 URL 붙여넣기 — _"방금 만든 게 지금 인터넷에 올라가 있어요"_

### 핵심 멘트

> "서버 설정 없었고, 배포 버튼 없었고, 명령어 하나였습니다."
> "아까 v0에서는 Publish 버튼 하나로 같은 결과를 냈죠. 도구만 다를 뿐, 결과는 같습니다."

## 슬라이드 8 — 비개발자 vs 개발자: 같은 결과, 다른 도구

> 개발자가 아니어도 괜찮습니다

| 레벨        | 조건                 | 추천 도구                                   | 배포 방법             | 오늘 시연        |
| ----------- | -------------------- | ------------------------------------------- | --------------------- | ---------------- |
| 비개발자    | 브라우저만 있으면 됨 | v0, Lovable, Bolt.new, Claude.ai            | Publish 딸깍          | v0 시연          |
| 입문 개발자 | 터미널 조금 써도 OK  | **Claude Code CLI**                         | `vercel` 명령어 한 줄 | Claude Code 시연 |
| 개발자      | 더 깊이 쓰고 싶다면  | Claude Code + MCP, Supabase, GitHub Actions | GitHub 연동 자동 배포 | Stitch MCP 시연  |

> 도구만 다를 뿐, "의도를 말하고 → AI가 만들고 → 내가 확인한다"는 동일합니다.

## 각주 — 발표 중 말로 풀어쓸 수 있는 상세 정보

[^1]:
    **v0 2026년 2월 대규모 업데이트**
    v0이 2026년 2월에 크게 업데이트되면서 단순 UI 생성 도구에서 프로덕션급 플랫폼으로 진화했습니다.
    추가된 기능: Git 연동, VS Code 스타일 에디터, 데이터베이스 연결, 에이전트 워크플로우.
    배포도 Publish 버튼 한 번이면 Vercel 인프라(글로벌 CDN, SOC 2 인증)에 바로 올라갑니다.
    → 멘트 예시: _"v0이 올해 2월에 대규모 업데이트가 있었는데요, 이제 VS Code처럼 코드도 직접 편집할 수 있고, DB도 붙일 수 있고, 배포는 Publish 버튼 누르면 끝입니다. 비개발자도 진짜 프로덕션 배포가 되는 거예요."_

[^2]:
    **Stitch AI 2026년 3월 19일 업데이트**
    Google의 Stitch가 3월 19일에 대규모 업데이트를 하면서 MCP 서버와 SDK를 공식 제공하기 시작했습니다.
    공식 엔드포인트 `stitch.googleapis.com/mcp`로 Claude Code, Gemini CLI, Cursor 등과 바로 연결됩니다.
    API 키는 stitch.withgoogle.com → 프로필 → Stitch settings → API key에서 발급합니다.
    → 멘트 예시: _"Stitch가 불과 9일 전에 공식 MCP 서버를 내놨습니다. 덕분에 Claude Code에서 '디자인 가져와줘' 한마디면 API로 직접 가져옵니다. 복붙이 아니라 자동화예요."_

[^3]:
    **MCP (Model Context Protocol) 란?**
    Anthropic이 만든 오픈 프로토콜로, AI 모델이 외부 도구/데이터에 표준화된 방식으로 접근할 수 있게 해줍니다.
    쉽게 말하면 "AI용 USB 포트" — 한 번 꽂으면 여러 도구를 자유롭게 쓸 수 있는 규격입니다.
    → 멘트 예시: _"MCP는 쉽게 말하면 AI용 USB 포트입니다. 예전에는 도구마다 연동 코드를 따로 짰는데, 이제는 MCP라는 표준 규격이 있어서 한 줄이면 연결됩니다."_

[^4]:
    **Vercel CLI 배포가 빠른 이유**
    Vercel은 프레임워크(Next.js, React 등)를 자동 감지하고 빌드 설정을 알아서 잡아줍니다.
    서버 설정, 도메인 연결, SSL 인증서, CDN 캐싱이 전부 자동입니다.
    → 멘트 예시: _"보통 웹사이트 배포하려면 서버 세팅하고, 도메인 연결하고, SSL 인증서 붙이고... 이런 작업이 필요한데요, Vercel은 이걸 전부 자동으로 해줍니다. 그래서 명령어 하나로 30초면 끝나는 거예요."_
