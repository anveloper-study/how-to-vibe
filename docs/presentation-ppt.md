## 슬라이드 2: 오늘의 목표

- 제목: 직접 만들고, 배포까지
- 핵심 문구: 코드를 직접 짜는 게 아니라 — 의도를 말하고, AI가 구현하고, 내가 확인하는 루프
- 오늘의 여정: 누구나 할 수 있는 방법(비개발자) → 개발자가 더 깊이 가는 방법

## 슬라이드 3: 바이브 코딩이란?

- 3원칙:
  - 01 의도 먼저: 기술 용어 없어도 됩니다. "파란 버튼 만들어줘"가 시작입니다.
  - 02 작게 이터레이션: 한 번에 완성하려 하지 마세요. 한 기능씩 확인하며 쌓으면 됩니다.
  - 03 결과물 책임은 나: AI가 짠 코드도 내 코드입니다. 실행해보고 이상하면 다시 시킵니다.
- 기존 개발과 다른 점: 타이핑 → 실행에서 대화 → 확인 루프로 전환

---

## 슬라이드 4: 프롬프트 잘 쓰는 법

- 공식: 맥락 + 원하는 동작 + 결과 형태
- 비교표:
  - "버튼 만들어줘" → "오른쪽 하단에 파란 CTA 버튼, 클릭 시 팀 매칭 시작되게 해줘"
  - "예쁘게 만들어줘" → "카드 컴포넌트에 hover 시 그림자 효과와 scale(1.02) 트랜지션 추가해줘"
  - "배포해줘" → "vercel로 프로덕션 배포하고 나온 URL 알려줘"
- 팁: 에러가 났을 때도 당황 금지 — 에러 메시지를 그대로 붙여넣으면 됩니다.
- 이 프롬프트 원칙은 비개발자 도구든 개발자 도구든 동일하게 적용됩니다.

## 슬라이드 5: DEMO — 비개발자 편: v0로 딸깍 배포

- 제목: 코드 없이, 브라우저만으로 배포까지
- 라이브 시연 순서:
  - 01 v0.app 접속
  - 02 프롬프트 입력: "팀 매칭 서비스 메인 페이지 만들어줘"
  - 03 결과 확인 → 수정 요청
  - 04 Publish 버튼 클릭 → 라이브 URL 생성
- 비개발자용 흐름도: 아이디어 → v0에 입력 → 결과 확인 → 수정 요청 → Publish → 라이브 URL
- 비개발자용 다른 도구들:
  - v0 by Vercel: 프롬프트로 UI 생성, Publish 버튼 딸깍 배포 (2026년 2월 대규모 업데이트)
  - Lovable: 자연어로 앱 생성 → 원클릭 배포, DB/인증도 자동 연결
  - Bolt.new: 브라우저 안에서 코드 편집 + 실시간 미리보기 + 배포까지
  - Claude.ai: 채팅으로 코드 생성 → 복사 붙여넣기로 활용
- 강조: 터미널을 한 번도 안 열었고, 코드도 한 줄도 안 썼는데 배포까지 끝났습니다.

## 슬라이드 6: 그런데 개발자는 더 깊이 갈 수 있습니다

- 제목: 비개발자 도구의 한계 → 개발자 도구로 확장
- 비개발자 도구로는 어려운 것들:
  - 세밀한 코드 제어와 커스터마이징
  - 외부 API/서비스 연동 (MCP)
  - 기존 프로젝트에 기능 추가
  - 복잡한 비즈니스 로직 구현
- 개발자 도구 파이프라인:
  - Google Stitch: 텍스트→UI 디자인 생성
  - stitch-mcp: MCP 브릿지, 디자인 전달
  - Claude Code: Next.js 앱 자동 변환
  - Vercel CLI: vercel 한 줄, 30초 배포
  - 라이브 URL: 전 세계에서 접근 가능

## 슬라이드 7: DEMO — 개발자 편 1: Stitch MCP

- 제목: Stitch MCP — 디자인 가져오기
- 4단계 순서:
  - 01 Stitch에서 Vibe Design으로 UI 생성
  - 02 Claude Code에 MCP 한 줄 등록
  - 03 디자인 가져오기 요청: "stitch에서 홈 화면 코드 가져와줘"
  - 04 Next.js 컴포넌트로 변환: "TypeScript + Tailwind 컴포넌트로 바꿔줘"
- MCP 등록 명령어: claude mcp add stitch --transport http https://stitch.googleapis.com/mcp --header "X-Goog-Api-Key: YOUR-API-KEY" -s user
- 강조: API 키 한 줄 등록으로 끝. DESIGN.md로 디자인 시스템도 자동으로 함께 가져옴.

## 슬라이드 8: DEMO — 개발자 편 2: Vercel CLI 배포

- 제목: Claude Code → Vercel 배포
- 4단계 순서:
  - 로컬 확인: npm run dev → 브라우저에서 미리보기
  - Vercel 배포: vercel → 프로젝트명 엔터 → 빌드 시작
  - URL 획득: https://xxx.vercel.app — 30초 안에 라이브 URL
  - 청중 공유: 방금 만든 게 지금 인터넷에 올라가 있어요
- 강조: 서버 설정 없었고, 배포 버튼 없었고, 명령어 하나였습니다.
- 비교: 아까 v0에서는 Publish 버튼로 같은 결과를 냈습니다. 도구만 다를 뿐, 결과는 같습니다.

## 슬라이드 9: 비개발자 vs 개발자 — 같은 결과, 다른 도구

- 제목: 스택 레벨별 진입로
- 비교표:
  - 비개발자 (브라우저만): v0, Lovable, Bolt.new → Publish 딸깍 배포 (오늘 v0 시연)
  - 입문 개발자 (터미널 OK): Claude Code CLI → vercel 명령어 한 줄 (오늘 Claude Code 시연)
  - 개발자 (더 깊이): Claude Code + MCP 확장, Supabase, GitHub Actions → 자동 배포 (오늘 Stitch MCP 시연)
- 핵심: 도구만 다를 뿐, "의도를 말하고 → AI가 만들고 → 내가 확인한다"는 동일
